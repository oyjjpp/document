# 微服务

微服务相关的主要知识点：

服务拆分：划分规则、划分策略等

服务设计模式：链式设计模式、聚合器设计模式、基于时间的异步模式等

服务间通信：一对一、一对多、微服务API

微服务治理：服务注册与发现、负载均衡、配置中心、容错机制、安全、链路追踪、熔断、限流、服务优雅上线等

微服务框架：go-zero、dubbo-go、go-micro、go-kit等

我在面试中被问的最多的就是：服务注册与发现、负载均衡、链路追踪、熔断、限流，这几个知识点是高频考点，尤其是负载均衡和限流，基本每场面试都问，建议大家把这一块好好看一下；

### rpc微服务框架
RPC（Remote Procedure Call）是一种远程过程调用协议，它允许不同的进程或计算机之间进行通信和调用远程函数或方法，就像本地调用一样。RPC框架提供了一种便捷的方式来实现这种通信，并隐藏了底层的通信细节。

微服务架构是一种将应用程序拆分成多个小型、自治的服务的方法。每个服务都专注于解决一个特定的业务问题，并且可以独立地进行开发、部署和扩展。微服务通常使用轻量级的通信协议进行通信，如HTTP或RPC。

因此，RPC微服务框架是一种特定的RPC框架，旨在支持微服务架构。它通常提供了更高级别的抽象，使得开发人员可以更容易地创建和管理微服务，同时提供了许多工具和功能，如服务注册与发现、负载均衡、故障恢复等，来支持微服务架构的核心原则：自治性和弹性。常见的RPC微服务框架包括gRPC、Apache Dubbo等。
